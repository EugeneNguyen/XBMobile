Boost up your incredible work, for free.
Start your UITableView with service support, cache support, pull to refresh, load more cell, dynamic height, asynchronize image loading, nested collectionView inside, and everything here. With just one library.
Following some simple steps, we can start and reuse everything with plist file only.
Getting started

With few steps below, you will jump to new incredible style of development

0. Install with CocoaPods

pod "XBMobile"
1. Import header file

#import <XBMobile.h>
2. Turn your UITableView to XBTableView

By code: 

XBTableView *tableView = ...
tableView.xbDelegate = self;
3. Prepare the plist

Full document of plist's properties here, and soon we will provide the IDE for XBMobile

4. Load & config table

//Load post params to request (optional)
tableView.postParams = @{@"token":[[NSUserDefaults standardUserDefaults] objectForKey:@"token"],
						 @"photo_id":@(123),
						 @"comment": @"This is a comment from XBTableView"};
[tableView loadInformationFromPlist:@"YOUR_PLIST_FILE_NAME"];
5. Using Delegate & extra function

// call when user tap to a row. item is the data stored with that index path
- (void)xbTableView:(XBTableView *)tableView didSelectRowAtIndexPath:(NSIndexPath *)indexPath forItem:(id)item;

// call when user tap to a button of a row. similar like the one above, but provide (UIButton *)btn so you can use it to separated the control.
- (void)xbTableView:(XBTableView *)tableView didSelectButton:(UIButton *)btn atIndexPath:(NSIndexPath *)indexPath forItem:(id)item;

// call at the end of normal cellForRowAtIndexPath:, with prepared cell and item, allow you to have the last chance to edit the cell before showing it. usually i do something like border colour or round corner, which isn't supported by XIB
- (UITableViewCell *)xbTableView:(XBTableView *)tableView cellForRowAtIndexPath:(NSIndexPath *)indexPath withPreparedCell:(UITableViewCell *)cell withItem:(id)item;

After all, we provide set of function & params to expand your options

// this is the information of plist. you can load it by function below, but if you have some plist's string somewhere, this is pretty useful for you.
@property (nonatomic, retain) NSDictionary *informations;

// Post params using in your query to fetch your data to the list
@property (nonatomic, retain) NSDictionary *postParams;

// allow you using HUB activity indicator and alert error when data from the server have some problem. Default of both is NO
@property (nonatomic, assign) BOOL usingHUD, usingErrorAlert;

@property (nonatomic, retain) IBOutlet id <XBTableViewDelegate> xbDelegate;

// Control to fetch all the data, including cache, data process, everything here.
@property (nonatomic, retain) XBDataFetching *dataFetching;

// load setting from plist
- (void)loadInformationFromPlist:(NSString *)plist;

// load data from custom array, when you don't use external data.
- (void)loadData:(NSArray *)data;
6. Start and enjoy the life :D
